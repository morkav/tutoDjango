Héritage de template

/!\ Regardez le code source avant /!\

L'héritage est une forme de réutilisation de template, mais ce n'est PAS une inclusion.

Dans cet exemple, la vue fait le rendu de CE (app6_inheritance.html) template.

Mais ce template hérite d'un autre template nommé app6_base.html.

Cela marche ainsi :

  1. This templates (app6_inheritance.html) says which template it inherits from using the extends tag.
  2. This templates define named blocks of content using the block tag.
  3. When the view renders the template, it wraps this (app6_inheritance.html) template in the parent (app6_base.html), filling the parent blocks with the child blocks.
  4. Ce template (app6_inheritance.html) dit de quel template il hérite en utilisant le tag extends.
  5. Ce template définie des blocs de contenu nommés avec le tag block.
  6. Quand la vue fait le rendu du template, elle enrobe ce (app6_inheritance.html) template dans le parent (app6_base.html), remplissant les blocks du parents avec ceux de l'enfant.

Schématiquement, cela marche ainsi :

Envant (app6_inheritance.html) :

         _____
        |_____|

         _____
        |     |
        |     |
        |_____|

au moment du rendu, il est enrobé DANS le parent (app6_base.html) :

    ................
    |     _____    |
    |    |_____|   |
    |              |
    |     _____    |
    |    |     |   |
    |    |     |   |
    |    |_____|   |
    |              |
    ................

Cela marche à l'envers d'une inclusion. Dans une inclusion, on ferait le rendu du parent, et on insérerait l'enfant.

Il n'est pas obligatoire de remplir tous les blocks du parent (qui peuvent avoir des valeurs par défaut), and on peut hériter d'un enfant, permettant ainsi des schémas d'héritage complexes.
